# 시연 스크립트 (발표 흐름 고정)

> 사전 준비: `web/dist`·`app/dist` 정적 서빙(또는 `npm run dev` 2개), 브라우저 탭 2개 열어두기.
> 인터넷이 불안하면 지도 배경만 안 뜨는 것이 정상 — 데이터 레이어는 모두 표시된다.

## 1. 문제 제기 → 입지 지도 (W1) — 1분
1. 대시보드 접속, **입지 지도** 탭 (기본 화면)
2. "제주에서만 연 2.6만 MWh 풍력이 버려진다" 멘트
3. **한림읍 셀 클릭** → 잉여 98점 팝업 ("풍력단지 옆인데 상권이 없다")
4. **이도2동 셀 클릭** → 상권·접근 100점 ("사람과 차는 도심에 있다")
5. "이 간극을 움직이는 배터리(EV)로 잇는 것이 허니콤"

## 2. 예측 + 최적화 → 스케줄 (W2) — 1분
1. **스케줄** 탭 클릭
2. 노란 area = 예측 잉여전력(2023-04-16, 실측 최대 제어일 1,354MWh)
3. 갈색 bar(충전)가 잉여 시간대에, 파란 bar(방전)가 저녁 피크에 배치됨을 지적
4. 상단 "LightGBM MAPE 5.9%" — 실데이터 검증 수치 강조

## 3. 확대 시나리오 (W3+W4) — 1분
1. **시나리오** 탭 클릭
2. 슬라이더 500 → 2,000으로 드래그: 절감액 3,160만원 → 1.24억원 실시간 갱신
3. 비교 테이블에서 "몇 대부터 경제성이 나오는지" 정부 의사결정 근거 멘트

## 4. 생성형 AI 브리핑 (W5) — 1분
1. **AI 브리핑** 탭 클릭 (슬라이더 상태 유지됨)
2. API 키 입력 → **브리핑 생성** 클릭 → 현재 시나리오 수치가 반영된 브리핑 생성
3. (네트워크 불안 시) 키 없이 클릭 → fallback 브리핑 표시됨을 그대로 시연해도 됨
   — "시연 환경 무관하게 항상 동작" 자체가 어필 포인트

## 5. 차주 경험 → 꿀차지 (M1~M3) — 1분
1. 앱 탭 전환 (모바일 프레임)
2. **홈**: "오늘 참여하면 +4,510원" + 꿀시간 하이라이트
3. **수익**: 누적 정산·방전량·CO₂ — "배터리 열화 보상이 목적함수에 반영됨" 멘트
4. **꿀 지갑**: 쿠폰·주차권 + 가맹점 지도 — "정부 대시보드의 입지 데이터를 그대로 재사용,
   잉여전력 → 차주 수익 → 지역 상권 낙수" 로 마무리

## 예상 질문 대비
- **"실차 제어는?"** → 데이터 기반 의사결정 지원 도구. 실증 하드웨어는 다음 단계
- **"태양광 제어량은?"** → KPX가 태양광 제어량을 별도 산정하지 않아 풍력 기준(보수적 추정)
- **"유휴 패턴 출처?"** → 실증 데이터 부재로 시간대·요일 기반 시뮬레이션(스펙 명시), 실증 시 교체
